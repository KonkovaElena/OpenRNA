# OpenRNA Code Of Conduct

## Purpose

This repository is intended to support rigorous engineering, scientific review, and honest technical discussion. Participation should make the project easier to evaluate, safer to contribute to, and more useful to people doing serious work.

## Expected Behavior

- Be precise about evidence, uncertainty, and implementation status.
- Challenge ideas with technical reasoning, not with personal attacks.
- Keep discussion focused on the repository, the science it cites, and the software it actually implements.
- Flag medical, regulatory, privacy, or security concerns early and concretely.
- Prefer reproducible examples, failing tests, logs, and primary sources over opinion alone.

## Unacceptable Behavior

- Harassment, intimidation, or discriminatory behavior.
- Personal insults, ridicule, or repeated bad-faith argument.
- Misrepresenting clinical, regulatory, or investor facts.
- Posting secrets, patient-identifying information, exploit details, or other sensitive material in public threads.
- Using the project issue tracker to request medical advice, diagnosis, or treatment recommendations.

## Scope

This code of conduct applies to repository discussions, issues, pull requests, reviews, and other project-managed communication channels.

## Enforcement

Project maintainers may edit, remove, lock, or close content that violates these expectations. Repeated or severe violations can result in restricted participation.

## Reporting

If the problem is behavioral, contact the repository owner through the GitHub profile associated with this repository.

If the problem is security-sensitive, use the process in [SECURITY.md](SECURITY.md) and avoid public disclosure.
