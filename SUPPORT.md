# Support

## Use This Repository For

- architecture and implementation review
- reproducible local setup questions
- test failures, CI failures, and bug reports
- release verification, SBOM, and provenance questions
- documentation corrections backed by primary sources

## Do Not Use This Repository For

- medical advice
- diagnosis, treatment selection, or patient-specific recommendations
- private vulnerability disclosure in public issues
- sponsor-confidential manufacturing or regulatory materials

## Where To Ask

- Usage and bug questions: GitHub issues via the structured templates in `.github/ISSUE_TEMPLATE/`
- Security-sensitive problems: follow [SECURITY.md](SECURITY.md)
- Technical publication or evidence corrections: open an issue with the source you want corrected

## Before Opening An Issue

1. Read [README.md](README.md).
2. Check [design.md](design.md) and the docs in `docs/`.
3. Reproduce on the current Node 24 baseline from `.nvmrc`.
4. Include commands, logs, expected behavior, and actual behavior.
